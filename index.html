1. Hash Commands

1.1. HEXPIRE

Syntax: HEXPIRE {key} {field} {seconds}

Function: Sets a time-to-live (TTL) on a specific field within a hash.

Return Value: Returns 1 if the timeout is set successfully, 0 if the key or field does not exist.

Notes: Currently, only the HEXPIRE command is supported for setting TTL on hash fields.

1.2. HEXPIREAT

Syntax: HEXPIREAT {key} {field} {timestamp}

Function: Sets an expiration time on a specific field in a hash, using an absolute UNIX timestamp.

Return Value: Returns 1 if the timeout is set successfully, 0 if the key or field does not exist.

Notes: Currently, only the HEXPIREAT command is supported for setting absolute expiration times on hash fields.

1.3. HEXPIRETIME

Syntax: HEXPIRETIME {key} {field}

Function: Returns the expiration time of a specific field in a hash as a UNIX timestamp.

Return Value: Returns the timestamp of the field's expiration, or -1 if the field does not have an expiration time.

Notes: Currently, only the HEXPIRETIME command is supported for querying the expiration time of hash fields.

1.4. HPERSIST

Syntax: HPERSIST {key} {field}

Function: Removes the expiration time from a field in a hash, making it persistent.

Return Value: Returns 1 if the timeout is removed, 0 if the key or field does not exist.

Notes: Currently, only the HPERSIST command is supported for removing expiration times on hash fields.

1.5. HPEXPIRE

Syntax: HPEXPIRE {key} {field} {milliseconds}

Function: Sets a time-to-live (TTL) on a specific field within a hash in milliseconds.

Return Value: Returns 1 if the timeout is set successfully, 0 if the key or field does not exist.

Notes: Currently, only the HPEXPIRE command is supported for setting TTL in milliseconds on hash fields.

1.6. HTTL

Syntax: HTTL {key} {field}

Function: Returns the TTL of a specific field in a hash in seconds.

Return Value: Returns the remaining TTL in seconds, or -2 if the key or field does not exist, or -1 if the field does not have an expiration time.

Notes: Currently, only the HTTL command is supported for querying the TTL in seconds of hash fields.

1.7. HRANDFIELD

Syntax: HRANDFIELD {key} [count] [WITHVALUES]

Function: Retrieves one or more random fields from a hash, optionally with their values.

Return Value: Returns the randomly selected field(s) or field-value pairs.

Notes: Currently, only the HRANDFIELD command is supported for retrieving random fields from a hash.

2. Set Commands

2.1. SMISMEMBER

Syntax: SMISMEMBER {key} {member} [member ...]

Function: Checks whether each specified member is a member of the set stored at the key.

Return Value: Returns an array of integers, with 1 for members that are in the set and 0 for those that are not.

Notes: Currently, only the SMISMEMBER command is supported for checking multiple memberships in a set.

3. ZSet Commands

3.1. ZDIFF

Syntax: ZDIFF {numkeys} {key} [key ...]

Function: Computes the difference between multiple sorted sets.

Return Value: Returns the members of the resulting sorted set.

Notes: Currently, only the ZDIFF command is supported for performing set difference operations on sorted sets.

3.2. ZDIFFSTORE

Syntax: ZDIFFSTORE {destination} {numkeys} {key} [key ...]

Function: Computes the difference between multiple sorted sets and stores the result in a destination key.

Return Value: Returns the number of elements in the resulting sorted set stored at the destination key.

Notes: Currently, only the ZDIFFSTORE command is supported for storing set differences into a sorted set.

4. Stream Commands

4.1. XGROUP CREATE

Syntax: XGROUP CREATE {stream} {groupname} {id or $} [MKSTREAM]

Function: Creates a consumer group in a stream.

Return Value: Returns OK if the group is created successfully.

Notes: Currently, only the XGROUP CREATE command is supported for creating consumer groups in streams.

4.2. XGROUP DESTROY

Syntax: XGROUP DESTROY {stream} {groupname}

Function: Removes a consumer group from a stream.

Return Value: Returns OK if the group is removed successfully.

Notes: Currently, only the XGROUP DESTROY command is supported for removing consumer groups from streams.

4.3. XGROUP SETID

Syntax: XGROUP SETID {stream} {groupname} {id}

Function: Sets the last delivered ID of a consumer group to a specific ID.

Return Value: Returns OK if the ID is set successfully.

Notes: Currently, only the XGROUP SETID command is supported for setting the ID of consumer groups in streams.

4.4. XPENDING

Syntax: XPENDING {stream} [groupname]

Function: Returns information about pending messages in a stream consumer group.

Return Value: Returns information about pending messages and their state.

Notes: Currently, only the XPENDING command is supported for querying pending messages in streams.

4.5. XINFO GROUPS

Syntax: XINFO GROUPS {stream}

Function: Retrieves information about consumer groups associated with a stream.

Return Value: Returns information about the consumer groups of the stream.

Notes: Currently, only the XINFO GROUPS command is supported for querying information about consumer groups.

4.6. XGROUP DELCONSUMER

Syntax: XGROUP DELCONSUMER {stream} {groupname} {consumername}

Function: Removes a consumer from a consumer group.

Return Value: Returns OK if the consumer is removed successfully.

Notes: Currently, only the XGROUP DELCONSUMER command is supported for removing consumers from a group.

4.7. XGROUP CREATECONSUMER

Syntax: XGROUP CREATECONSUMER {stream} {groupname} {consumername}

Function: Creates a consumer within a consumer group.

Return Value: Returns OK if the consumer is created successfully.

Notes: Currently, only the XGROUP CREATECONSUMER command is supported for adding consumers to a group.

5. BloomFilter Commands

5.1. BF.LOADCHUNK

Syntax: BF.LOADCHUNK {key} {chunk}

Function: Loads a chunk of data into a Bloom filter.

Return Value: Returns OK if the chunk is loaded successfully.

Notes: Currently, only the BF.LOADCHUNK command is supported for loading data into Bloom filters.

5.2. BF.SCANDUMP

Syntax: BF.SCANDUMP {key} {iterator}

Function: Scans and dumps the contents of a Bloom filter.

Return Value: Returns a serialized dump of the Bloom filter's contents.

Notes: Currently, only the BF.SCANDUMP command is supported for dumping Bloom filter data.

6. JSON Commands

6.1. JSON.DEBUG MEMORY

Syntax: JSON.DEBUG MEMORY {key}

Function: Provides memory usage statistics for a JSON document.

Return Value: Returns memory usage statistics for the specified JSON document.

Notes: Currently, only the JSON.DEBUG MEMORY command is supported for analyzing JSON document memory usage.

7. Search Commands

7.1. FT.AGGREGATE

Syntax: FT.AGGREGATE {index} {query}

Function: Performs aggregation queries on search indexes.

Return Value: Returns aggregated results based on the query.

Notes: Currently, only the FT.AGGREGATE command is supported for aggregation queries.

7.2. FT.ALIASADD

Syntax: FT.ALIASADD {alias} {index}

Function: Adds an alias for a search index.

Return Value: Returns OK if the alias is added successfully.

Notes: Currently, only the FT.ALIASADD command is supported for adding aliases to search indexes.

7.3. FT.ALIASDEL

Syntax: FT.ALIASDEL {alias}

Function: Deletes an alias for a search index.

Return Value: Returns OK if the alias is deleted successfully.

Notes: Currently, only the FT.ALIASDEL command is supported for removing aliases from search indexes.

7.4. FT.ALIASUPDATE

Syntax: FT.ALIASUPDATE {alias} {index}

Function: Updates an alias to point to a different search index.

Return Value: Returns OK if the alias is updated successfully.

Notes: Currently, only the FT.ALIASUPDATE command is supported for updating search index aliases.

7.5. FT.ALTER

Syntax: FT.ALTER {index} {options}

Function: Alters the configuration of a search index.

Return Value: Returns OK if the index is altered successfully.

Notes: Currently, only the FT.ALTER command is supported for altering search index configurations.

7.6. FT.CONFIG GET

Syntax: FT.CONFIG GET {name}

Function: Retrieves the configuration of a search index or Redisearch module.

Return Value: Returns the configuration details.

Notes: Currently, only the FT.CONFIG GET command is supported for retrieving configuration settings.

7.7. FT.CONFIG SET

Syntax: FT.CONFIG SET {name} {value}

Function: Sets the configuration of a search index or Redisearch module.

Return Value: Returns OK if the configuration is set successfully.

Notes: Currently, only the FT.CONFIG SET command is supported for setting configuration values.

7.8. FT.CURSOR DEL

Syntax: FT.CURSOR DEL {cursor}

Function: Deletes a search cursor.

Return Value: Returns OK if the cursor is deleted successfully.

Notes: Currently, only the FT.CURSOR DEL command is supported for deleting search cursors.

7.9. FT.CURSOR READ

Syntax: FT.CURSOR READ {cursor}

Function: Reads data from a search cursor.

Return Value: Returns the data read from the cursor.

Notes: Currently, only the FT.CURSOR READ command is supported for reading from search cursors.

7.10. FT.DICTADD

Syntax: FT.DICTADD {dict} {word} [word ...]

Function: Adds words to a search dictionary.

Return Value: Returns OK if the words are added successfully.

Notes: Currently, only the FT.DICTADD command is supported for managing search dictionaries.

7.11. FT.DICTDEL

Syntax: FT.DICTDEL {dict} {word} [word ...]

Function: Deletes words from a search dictionary.

Return Value: Returns OK if the words are deleted successfully.

Notes: Currently, only the FT.DICTDEL command is supported for removing words from search dictionaries.

7.12. FT.DICTDUMP

Syntax: FT.DICTDUMP {dict}

Function: Dumps the contents of a search dictionary.

Return Value: Returns a serialized dump of the dictionary.

Notes: Currently, only the FT.DICTDUMP command is supported for dumping search dictionary contents.

7.13. FT.EXPLAINCLI

Syntax: FT.EXPLAINCLI {query}

Function: Explains a search query.

Return Value: Returns an explanation of the query execution.

Notes: Currently, only the FT.EXPLAINCLI command is supported for explaining search queries.

7.14. FT.PROFILE

Syntax: FT.PROFILE {index} {query}

Function: Profiles the execution of a search query.

Return Value: Returns profiling information about the query.

Notes: Currently, only the FT.PROFILE command is supported for profiling search queries.

7.15. FT.SPELLCHECK

Syntax: FT.SPELLCHECK {index} {query}

Function: Performs spell checking on a search query.

Return Value: Returns spelling suggestions based on the query.

Notes: Currently, only the FT.SPELLCHECK command is supported for spell checking search queries.

7.16. FT.SYNDUMP

Syntax: FT.SYNDUMP {index}

Function: Dumps the contents of a synonym dictionary.

Return Value: Returns a serialized dump of the synonym dictionary.

Notes: Currently, only the FT.SYNDUMP command is supported for dumping synonym dictionaries.

7.17. FT.SYNUPDATE

Syntax: FT.SYNUPDATE {index} {synonym}

Function: Updates a synonym dictionary in a search index.

Return Value: Returns OK if the dictionary is updated successfully.

Notes: Currently, only the FT.SYNUPDATE command is supported for updating synonym dictionaries.

7.18. FT.TAGVALS

Syntax: FT.TAGVALS {index} {tag}

Function: Retrieves all tag values for a given tag in a search index.

Return Value: Returns a list of tag values.

Notes: Currently, only the FT.TAGVALS command is supported for retrieving tag values.

8. HyperLogLog Commands

8.1. PFSELFTEST

Syntax: PFSELFTEST

Function: Performs a self-test on the HyperLogLog data structure to check its integrity.

Return Value: Returns OK if the test is successful, otherwise returns an error.

Notes: Currently, only the PFSELFTEST command is supported for testing HyperLogLog integrity.

8.2. PFDEBUG

Syntax: PFDEBUG {subcommand} [arguments ...]

Function: Provides debugging information about the HyperLogLog data structure.

Return Value: Returns debugging information based on the subcommand and arguments.

Notes: Currently, only the PFDEBUG command is supported for debugging HyperLogLog data structures.
